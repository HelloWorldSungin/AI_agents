# Dual System Workflow: AI_agents + Taches-CC-Resources

This guide explains how to effectively use both the AI_agents composed system and the Taches-CC-Resources tools together.

## Overview

The AI_agents repository contains two complementary systems:

| System | Location | Purpose |
|--------|----------|---------|
| **AI_agents Composed** | `.ai-agents/composed/` | Multi-agent software development |
| **Taches-CC-Resources** | `external/taches-cc-resources/` | Claude Code extension development |

These systems solve different problems and should be used for their respective strengths.

## System Comparison

### AI_agents Composed Agents

Role-based developer agents for building software applications:

- `backend_developer.md` - Backend development tasks
- `frontend_developer.md` - Frontend development tasks
- `team_manager.md` - Orchestration and coordination
- `qa_tester.md` - Quality assurance and testing
- `minimal_developer.md` - Lightweight software developer
- `frontend_specialist.md` - Specialized frontend work

**Characteristics:**
- Large comprehensive prompts (1000+ lines)
- Generated by `compose_agent.py` script
- Include embedded Anthropic skills
- Multi-agent coordination protocols
- Self-contained, no external dependencies

### Taches-CC-Resources

Specialized tools for Claude Code extension development:

**Skills:**
- `create-agent-skills` - Create Claude Code skills
- `create-slash-commands` - Create slash commands
- `create-subagents` - Create subagent configurations
- `create-hooks` - Create Claude Code hooks
- `create-plans` - Create project plans
- `debug-like-expert` - Deep debugging methodology

**Auditors:**
- `skill-auditor` - Validate skills against best practices
- `slash-command-auditor` - Validate slash commands
- `subagent-auditor` - Validate subagent configurations

**Characteristics:**
- Focused single-purpose tools
- Pure XML structure with YAML frontmatter
- Designed for Claude Code's Task tool
- Reference external skills for best practices

## Practical Workflow Example

### Full Development Cycle

```
1. SOFTWARE DEVELOPMENT (AI_agents Composed)
   ├── Use composed agents to build your app
   ├── team_manager orchestrates work distribution
   ├── frontend_developer handles UI components
   ├── backend_developer handles API/data layer
   └── qa_tester validates the implementation

2. TOOLING DEVELOPMENT (Taches-CC-Resources)
   ├── Notice repetitive patterns in your workflow
   ├── Create a skill to automate it:
   │   └── skill: create-agent-skills
   ├── Create a slash command for quick access:
   │   └── /create-slash-command
   └── Audit before committing:
       └── /audit-skill ./skills/my-skill/SKILL.md

3. ITERATE
   └── Your new skill/command speeds up step 1
```

### Quick Reference Commands

```bash
# ═══════════════════════════════════════════════════════════════
# SOFTWARE BUILDING (AI_agents Composed)
# ═══════════════════════════════════════════════════════════════

# Generate a composed agent
cd your-project
python ~/GIT/AI_agents/scripts/compose_agent.py

# Run orchestration with composed agents
python ~/GIT/AI_agents/scripts/orchestration/run_team.py

# ═══════════════════════════════════════════════════════════════
# CLAUDE CODE TOOLING (Taches-CC-Resources)
# ═══════════════════════════════════════════════════════════════

# Create new Claude Code extensions

/create-slash-command         # Create a slash command

# Audit before shipping
/audit-skill <path>           # Validate a skill
/audit-slash-command <path>   # Validate a slash command
/audit-subagent <path>        # Validate a subagent

# Planning and debugging
/create-plan [description]    # Plan a project
/debug [issue]                # Debug methodology
```

## Integration Architecture

The two systems are integrated via the `external/` submodule structure:

```
AI_agents/
├── .ai-agents/
│   └── composed/           # AI_agents composed agents
├── .claude/
│   ├── commands/           # Slash command wrappers
│   └── skills/             # Symlinks to taches skills
├── external/
│   ├── anthropic-skills/   # Official Anthropic skills
│   └── taches-cc-resources/# Taches tools (submodule)
└── scripts/
    ├── compose_agent.py    # Agent composition script
    └── setup-commands.py   # Tool installation script
```

### Installing Taches Tools to Other Projects

```bash
# Install all taches tools to another project
python scripts/setup-commands.py /path/to/other-project

# Install globally
python scripts/setup-commands.py --global

# List available tools without installing
python scripts/setup-commands.py --list
```

## Best Practices

### Do Use AI_agents Composed When:
- Building complete software applications
- Need multi-agent coordination
- Working on frontend/backend/QA tasks
- Want self-contained agent definitions

### Do Use Taches-CC-Resources When:
- Creating Claude Code skills, commands, or subagents
- Need to audit/validate Claude Code artifacts
- Want focused single-purpose tools
- Planning projects or debugging complex issues

### Don't:
- Mix the two systems' formats (XML vs markdown composition)
- Copy taches agents into AI_agents composed (different dependencies)
- Use AI_agents composed for Claude Code extension development
- Use taches auditors for non-Claude-Code artifacts

## Summary

The two systems are **complementary**:

- **AI_agents Composed** builds software applications
- **Taches-CC-Resources** builds the tools that make development faster

Use each system for its intended purpose, and leverage the integration via `external/` submodules and `.claude/skills/` symlinks to access both seamlessly.
